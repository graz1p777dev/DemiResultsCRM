# Концепция A — «Command Center»

## Идея

Dashboard как **пульт управления компанией в реальном времени**.

Вдохновение: Bloomberg Terminal, Notion Dashboard, Linear.

Всё на одном экране. Никаких прокруток по вертикали (или минимум).
Владелец видит полную картину дня без единого клика.

## Блоки и их роль

| Зона | Блок | Высота | Приоритет |
|------|------|--------|-----------|
| Верх | 5 KPI-карточек в ряд | 120px | Критический |
| Середина лево | Таблица план vs факт | 220px | Критический |
| Середина право | Таблица сотрудников | 220px | Высокий |
| Низ лево | Последние записи | 180px | Средний |
| Низ центр | Финансы месяца | 180px | Средний |
| Низ право | Активность сегодня | 180px | Дополнительный |

## KPI первой строки

```
1. Выручка месяц — главный показатель бизнеса
2. ФВ сегодня — активность прямо сейчас
3. Продажи месяц — конверсия в деньги
4. Средний чек — качество сделок
5. Прибыль месяц — итог всего
```

## Почему эта структура подходит Demi Results OS

✅ **Полная картина без прокрутки** — владелец видит всё за 5 секунд

✅ **Иерархия по важности** — деньги сверху, детали снизу

✅ **Real-time чувство** — "ФВ сегодня 3/5" показывает текущий момент

✅ **Сотрудники на виду** — сразу видно кто работает, кто нет

❌ **Минус** — плотный, требует привыкания. Не для мобильных.

# Концепция B — «Morning Brief»

## Идея

Dashboard как **ежедневный брифинг для владельца**.

Вдохновение: Stripe Dashboard, Notion Weekly Overview.

Структура читается сверху вниз как история:
"Вот деньги → вот как идёт план → вот кто работает → вот детали".

Больше воздуха, карточки крупнее, данные читаются легче.
Идеально для тех, кто открывает Dashboard утром.

## KPI первого блока

```
1. Выручка месяц — с прогресс-баром и % к прошлому месяцу
2. Прибыль месяц — рентабельность в %
3. ФВ (встречи) — месяц + сегодняшний день
4. Продажи — месяц + средний чек
```

## Почему эта структура подходит Demi Results OS

✅ **Читается как история** — логичный поток сверху вниз

✅ **Карточки сотрудников** — каждый человек виден отдельно, не в таблице

✅ **Приветствие с контекстом** — владелец сразу понимает ситуацию

✅ **Просторный дизайн** — меньше усталости при ежедневном использовании

✅ **Легче для новых сотрудников** — РОП или МП тоже поймут без обучения

❌ **Минус** — требует прокрутки. Не всё видно сразу.

# Концепция C — «Split Focus»

## Идея

Dashboard разделён на **две независимые зоны**.

Вдохновение: Figma, Linear, Jira — двухпанельный интерфейс.

**Левая панель** — стабильные метрики месяца (не меняются каждый день).
**Правая панель** — динамика сегодняшнего дня (обновляется в реальном времени).

Владелец использует левую панель для стратегических решений,
правую — для оперативного контроля.

## KPI

```
Левая панель (месячные):
1. Выручка / Расход / Прибыль
2. ФВ план vs факт
3. Продажи план vs факт
4. KPI каждого сотрудника

Правая панель (сегодняшние):
1. ФВ сегодня
2. Продажи сегодня
3. Лента последних событий
4. Расписание оставшегося дня
```

## Почему эта структура подходит Demi Results OS

✅ **Разделение горизонтов** — месяц и день не смешиваются

✅ **Лента событий** — ощущение живого контроля, как Slack/Telegram

✅ **Всё видно без прокрутки** — две панели покрывают всё

✅ **Масштабируемость** — правая панель легко расширяется

✅ **Идеально для растущей команды** — можно добавить 3й, 4й сотрудника

❌ **Минус** — непривычная структура для первого знакомства

# Рекомендация

Для Demi Results OS на текущем этапе наиболее подходит **Концепция A (Command Center)** или **Концепция C (Split Focus)**.

**Концепция A** — если приоритет: "Открыл и за 5 секунд понял всё".

**Концепция C** — если приоритет: "Разделить стратегию (месяц) и тактику (сегодня)".

**Концепция B** — если в будущем появится мобильное приложение и нужна единая логика.
