## 1. Философия дизайна

Дизайн Demi Results OS строится на одном принципе:

**Данные — главный герой. Интерфейс — инструмент.**

Никаких украшений ради украшений.
Каждый элемент интерфейса существует для того, чтобы владелец принял решение быстрее.

### Три кита дизайна

| Принцип | Что это значит |
|---------|---------------|
| **Clarity** — Ясность | Пользователь понимает, где он и что делать, за 3 секунды |
| **Density** — Плотность | Максимум информации на экране без перегрузки |
| **Trust** — Доверие | Система выглядит надёжно. Данные — точно. Цвета — спокойно |

## 3. Типографика

### Шрифт

**Основной шрифт:** `Inter` (Google Fonts)

Причина выбора: максимальная читаемость данных и цифр на любом разрешении.

### Типографическая шкала

| Роль | Размер | Вес | Применение |
|------|--------|-----|------------|
| **Display** | 32px | 700 | Заголовок страницы (Dashboard title) |
| **H1** | 24px | 700 | Заголовки разделов |
| **H2** | 18px | 600 | Заголовки карточек, панелей |
| **H3** | 15px | 600 | Подзаголовки, названия метрик |
| **Body** | 14px | 400 | Основной текст, таблицы |
| **Small** | 12px | 400 | Подписи, метки, вспомогательный текст |
| **Micro** | 10px | 500 | Теги, badges, единицы измерения |

### Числа и метрики

Все ключевые метрики (KPI, суммы, проценты) — шрифт `tabular-nums` для выравнивания колонок.

## 4. Сетка и Layout

### Основной Layout

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  SIDEBAR (240px)  │  MAIN CONTENT (flex-1)          │
│                   │                                 │
│  Навигация        │  Header (60px)                  │
│  Профиль          │  ─────────────────────────────  │
│                   │  Content Area (scrollable)      │
│                   │                                 │
└─────────────────────────────────────────────────────┘
```

### Контентная сетка

- **Карточки метрик:** 4 колонки (desktop) → 2 (tablet) → 1 (mobile)
- **Таблицы:** full-width с горизонтальным скроллом
- **Боковые панели:** max-width 380px
- **Модальные окна:** max-width 480px (обычные), 720px (расширенные)

### Отступы (Spacing Scale)

```
4px   → xs   (иконки внутри кнопки)
8px   → sm   (padding внутри badge)
12px  → md   (gap между элементами)
16px  → lg   (padding карточки)
24px  → xl   (gap между карточками)
32px  → 2xl  (padding страницы)
48px  → 3xl  (разделители секций)
```

## 6. Иконки

Причина: минималистичный стиль, полное покрытие бизнес-иконок, совместимость с shadcn/ui.

```

20px → стандартные (в навигации, кнопках)
```

## 7. Анимации и переходы

```
Стандартный переход:  150ms ease
Появление карточек:   250ms ease-out (fadeInUp)
Открытие модала:      200ms ease-out
Закрытие модала:      150ms ease-in
Hover эффекты:        150ms ease
Skeleton loading:     1.5s shimmer loop

Принцип: анимации не должны задерживать работу.
Если пользователь торопится — интерфейс не должен его ждать.
```

## 8. Dashboard — Проект первого экрана

### Концепция

Dashboard — это **пульт управления компанией**.

Владелец открывает его и за 5–10 секунд понимает:
- Как идёт день сегодня?
- Где отставание от плана?
- Кто из сотрудников не выполняет показатели?
- Какова финансовая ситуация?

### Принципы Dashboard

**Иерархия информации:**
1. Деньги (выручка, прибыль) — самый верх
2. Активность (ФВ, продажи) — рядом
3. Люди (сотрудники, их показатели) — средняя зона
4. Детали (записи, события) — нижняя зона

**Цветовые сигналы на Dashboard:**
- Зелёный прогресс-бар → идём по плану (≥80%)
- Оранжевый → отстаём (50–79%)
- Красный → тревога (<50%)

**Пустые состояния:**
- Нет данных за сегодня → показывается заглушка с иконкой и текстом "Данных пока нет"
- Нет сотрудников → кнопка "Добавить сотрудника"

## 9. Ролевой дизайн

Интерфейс адаптируется под роль пользователя.

| Роль | Что видит на Dashboard |
|------|------------------------|
| **Владелец** | Все блоки: финансы, все сотрудники, полная аналитика |
| **МП** | Только свои показатели: ФВ, продажи, выручка за день/месяц |
| **LMAI** | Только свои показатели: обращения, лиды, НВ |
| **РОП** | Все сотрудники + их планы, без финансов |

## 11. Адаптивность

| Breakpoint | Ширина | Поведение |
|------------|--------|-----------|
| Desktop | ≥1280px | Полный layout, sidebar fixed |
| Laptop | 1024–1279px | Sidebar fixed, контент сжимается |
| Tablet | 768–1023px | Sidebar скрывается, появляется hamburger |
| Mobile | <768px | Только основные метрики, табличная навигация снизу |

**Приоритет:** система проектируется для Desktop.
Мобильная версия — второй этап разработки.

---

## 12. Доступность (Accessibility)

- Все интерактивные элементы доступны с клавиатуры
- Focus-ring: `2px solid #0c4d6c`
- Contrast ratio: минимум 4.5:1 для текста
- Все иконки сопровождаются `aria-label`
- Escape закрывает любое модальное окно

---

## 13. Что НЕ делаем в дизайне

- Не используем анимации ради анимаций
- Не делаем тёмный sidebar светлым без причины
- Не добавляем цвета вне фирменной палитры
- Не делаем кнопки без чёткого действия
- Не прячем важные данные за лишними кликами
- Не используем более 3 цветов одновременно в одном блоке
