## Фаза 2: Левая панель — Данные за месяц

### Задача 2.1 — Функции получения данных за месяц

**Что будет сделано:**
Создать `src/lib/dashboard-queries.ts` — чистые функции для запросов к Supabase.
Все функции — async, принимают Supabase client, возвращают типизированные данные.

Функции для левой панели:
```typescript
getMonthKpiStats(supabase, year, month, employeeId?)
  → { fv: number, sales: number, revenue: number, kpiPct: number }

getMonthPlanStats(supabase, year, month, employeeId?)
  → { planFv: number, planSales: number, planRevenue: number }

getRevenueByWeeks(supabase, year, month)
  → Array<{ week: string, revenue: number }>

getPlanVsFactTable(supabase, year, month)
  → Array<{ employeeId, name, planFv, factFv, planSales, factSales, planRevenue, factRevenue, kpiPct }>
```

Каждая функция:
- Обрабатывает `error` от Supabase (бросает или возвращает null-safe значения)
- Адаптирована к **реальной схеме БД** (определённой в задаче 0.1)

**Какие файлы изменяются:**
- `src/lib/dashboard-queries.ts` — создание

**Supabase таблицы:**
- `consultations` (fv=count пришедших, sales=count купивших, revenue=sum amount)
- `daily_facts` (если существует — используем для override)
- `decomposition` (если существует — агрегат)
- `employee_kpi` (планы)
- `employees` (список с именами)

**Зависимости:** Задача 0.1 (знаем реальную схему), 0.2 (типы)

**Критерии готовности:**
- [ ] Все функции типизированы, нет `any`
- [ ] Ошибки Supabase обрабатываются
- [ ] Функции работают при пустой БД (возвращают нули, не падают)
- [ ] Каждая функция — отдельный `await`, можно вызывать через `Promise.all`
