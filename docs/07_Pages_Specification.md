## Глоссарий сокращений

- **ФВ** — первичная встреча с клиентом
- **МП** — менеджер по продажам
- **LMAI** — менеджер по лидогенерации / маркетингу
- **РОП** — руководитель отдела продаж
- **НВ** — новый входящий клиент (без предварительной записи)
- **KPI** — ключевой показатель эффективности
- **Декомпозиция** — детальный план и факт по сотруднику

# 1. Dashboard

## 1.1 Цель страницы

Дать владельцу полный срез состояния бизнеса за 5–10 секунд.
Заменяет ежедневные совещания, сводные таблицы и утренние отчёты.

## 1.2 Доступ

| Роль | Что видит |
|------|-----------|
| Владелец | Всё: финансы, вся команда, все метрики |
| РОП | Команда + планы (без финансов и прибыли) |
| МП | Только свои показатели за день и месяц |
| LMAI | Только свои показатели: лиды, обращения, НВ |

## 1.3 Layout

Концепция **Split Focus** (утверждена):
- Sidebar (nav) → Left Panel (месяц) → Right Panel (сегодня)

## 1.4 Левая панель — Месяц

### Блок: Прогресс месяца
- Название месяца + год
- Сколько дней прошло из 30/31
- Прогресс-бар: X из Y рабочих дней

### Блок: Финансы (только Владелец и Бухгалтер)
- Прибыль месяца — главная цифра (крупно)
- Доход / Расход рядом
- Рентабельность в %
- Мини-chart: выручка по неделям (Bar Chart, 4 недели)

### Блок: План vs Факт
Таблица без границ:
- ФВ: факт / план · % · прогресс-бар
- Продажи: факт / план · % · прогресс-бар
- Выручка: факт / план · % · прогресс-бар
- Лиды (LMAI): факт / план · %
- Обращения: факт / план · %

Цветовая индикация: <50% красный, 50–79% жёлтый, ≥80% синий/зелёный.

### Блок: KPI команды
Строки по каждому сотруднику:
- Аватар + Имя + Роль
- Мини прогресс-бар KPI
- KPI % (цветной)
- Выручка (только для МП)
- Статус: на месте / не вышел

## 1.5 Правая панель — Сегодня

### Блок: Карточки дня
- ФВ сегодня: факт / план · прогресс-бар · % badge
- Продажи сегодня: факт / план · выручка дня

### Блок: Лента событий (Live Feed)
Хронологический список событий за сегодня:
- Время · Цветной индикатор · Имя клиента · Менеджер · Статус · Сумма
- Статусы: Купила / Не купила / Дожать / Не пришла / Предоплата
- Новые события появляются наверху
- Отображается последние 8–10 событий
- Ссылка "Все записи →"

### Блок: Команда сейчас
- Аватар + Имя + Роль + Статус (на месте / в процессе ФВ)
- ФВ / Продажи / KPI для каждого

### Блок: Расписание (остаток дня)
- Время · Имя клиента · Тип (ФВ / Запись)
- Только предстоящие события
- Ссылка в раздел Записей

## 1.6 Данные

| Данные | Источник |
|--------|----------|
| Финансы | таблица `finances` |
| ФВ, Продажи | таблица `consultations` |
| Лиды, Обращения | таблица `daily_facts` |
| KPI сотрудников | расчёт из `kpi_plans` + `consultations` |
| Расписание | `consultations` WHERE date = today |
| Статус сотрудников | `attendance` |

## 1.7 Автоматизация

- При загрузке Dashboard — запрос свежих данных (no cache)
- Автообновление данных каждые 5 минут
- Уведомление если сотрудник не отметился к 10:00

## 1.8 Связи с другими страницами

- Клик на сотрудника → страница Декомпозиции этого сотрудника
- "Все записи →" → Записи на консультацию
- Финансовый блок → Финансы
- Клик на KPI → Зарплаты

# 2. Финансы

## 2.1 Цель страницы

Полный финансовый учёт компании: доходы, расходы, прибыль, кассовые разрывы.
Заменяет Excel-таблицы бухгалтера и Google Sheets владельца.

## 2.2 Доступ

Владелец (полный), Бухгалтер (полный).

## 2.3 Внутренние страницы

### 2.3.1 /finances — Обзор

**Топ-строка (4 карточки):**
- Доход месяца
- Расход месяца
- Прибыль месяца
- Рентабельность %

**График: Доходы vs Расходы по месяцам**
- Bar Chart · 6 последних месяцев
- Две серии: Доход (navy) / Расход (steel)
- Hover tooltip: детали месяца

**Таблица последних транзакций:**
- Дата · Категория · Описание · Сумма · Тип (Доход/Расход) · Кто добавил
- Сортировка по дате (новые сверху)
- Пагинация: 20 записей на странице
- Фильтры: Тип · Категория · Период · Сумма

### 2.3.2 /finances/income — Доходы

**Блоки:**
- Итого за выбранный период
- Разбивка по категориям (таблица + мини-пайчарт)
- Полная таблица доходов

**Категории доходов:**
- Продажи (из записей на консультацию — автоматически)
- Инвестиции / Вклады владельца
- Прочие поступления

### 2.3.3 /finances/expenses — Расходы

**Блоки:**
- Итого расходов за период
- Топ категорий расходов (horizontal bar)
- Полная таблица расходов

**Категории расходов:**
- Зарплата (из модуля Зарплаты — автоматически)
- Аренда
- Маркетинг / Реклама
- Доставка
- Коммунальные
- Прочее

### 2.3.4 /finances/add — Добавить транзакцию

Форма (Modal или отдельная страница):
- Тип: Доход / Расход (toggle)
- Категория (Select)
- Сумма
- Дата
- Описание
- Прикрепить документ (опционально)
- Кто вносит (автоматически = текущий пользователь)

## 2.4 Действия пользователя

- Добавить доход / расход
- Редактировать транзакцию
- Удалить транзакцию (с подтверждением)
- Экспорт в Excel за период
- Фильтровать по периоду, категории, типу

## 2.5 Данные

| Данные | Источник |
|--------|----------|
| Транзакции | таблица `finances` |
| Продажи | автоматически из `consultations` WHERE status_after_fb IN ('Купила', 'Предоплата') |
| Зарплаты | автоматически из `salaries` |

## 2.6 Автоматизация

- При сохранении записи "Купила" — автоматически создаётся транзакция дохода в `finances`
- При расчёте зарплаты — автоматически создаётся транзакция расхода
- Ежемесячный итог считается автоматически

## 2.7 Связи

- Dashboard ← итоги финансов
- Записи → авто-транзакции при продаже
- Зарплаты → авто-расход при выплате

# 3. Зарплаты

## 3.1 Цель страницы

Автоматический расчёт зарплаты каждого сотрудника на основе:
- оклада,
- выполнения KPI,
- бонусов,
- штрафов,
- рабочих дней (по расписанию и явкам).

## 3.2 Доступ

Владелец (полный), Бухгалтер (полный), Сотрудник (только своя зарплата).

## 3.3 Внутренние страницы

### 3.3.1 /salaries — Обзор

**Период:** выбор месяца (по умолчанию — текущий)

**Таблица сотрудников:**
- Имя · Должность · Оклад · KPI% · Бонус · Штраф · К выплате · Статус (Выплачена / Ожидает)
- Клик на строку → детальная страница сотрудника

**Итоговая строка:**
- ФОТ (фонд оплаты труда) за месяц
- Выплачено / Ожидает выплаты

**Кнопки:**
- "Рассчитать все зарплаты" (только Владелец)
- "Экспорт в Excel"
- "Отметить как выплачено" (bulk action)

### 3.3.2 /salaries/:employee_id — Зарплата сотрудника

**Заголовок:** Имя + Должность + Месяц

**Блок 1: Расчёт**
- Оклад (базовый)
- KPI бонус: расчёт по формуле + % выполнения
- Индивидуальные бонусы (ручные)
- Штрафы (с причиной)
- Итого к выплате

**Блок 2: Детали KPI**
- Таблица по метрикам: план · факт · % · вес в KPI
- ФВ, Продажи, Выручка (для МП)
- Лиды, Обращения, НВ (для LMAI)

**Блок 3: Рабочие дни**
- Сколько дней отработано / план
- Сколько больничных / отгулов / неявок
- Таблица с явками (из Рабочего календаря)

**Блок 4: История выплат**
- Предыдущие месяцы: дата выплаты · сумма · статус

**Действия:**
- Добавить бонус (с комментарием)
- Добавить штраф (с причиной)
- Отметить зарплату выплаченной
- Распечатать расчётный лист

## 3.4 Данные

| Данные | Источник |
|--------|----------|
| Оклад | `employees.salary` |
| KPI план | `kpi_plans` |
| KPI факт | `consultations` + `daily_facts` |
| Явки | `attendance` |
| Бонусы/штрафы | `salary_adjustments` |
| Расписание | `schedules` |

## 3.5 Автоматизация

- При закрытии месяца — автоматически генерируется расчёт для всех сотрудников
- При отметке "Выплачена" — создаётся транзакция расхода в Финансах
- Уведомление Владельцу: "Зарплаты за [месяц] готовы к выплате"

## 3.6 Формула расчёта KPI-бонуса

```
Бонус = Оклад × KPI_коэффициент × (факт_KPI / план_KPI)

KPI_коэффициент — задаётся в настройках для каждой роли.
Если факт < минимального порога (напр. 30%) — бонус = 0.
Если факт ≥ 100% — применяется повышающий коэффициент.
```

## 3.7 Связи

- Рабочий календарь → явки → рабочие дни
- Декомпозиция → KPI факт
- Финансы → транзакция выплаты

# 4. Декомпозиция

## 4.1 Цель страницы

Ежедневный план-факт каждого сотрудника на текущий месяц.
Владелец видит все. Сотрудник видит только свою декомпозицию.

Это основная операционная страница — открывается каждый день.

## 4.2 Доступ

Владелец (все), РОП (своя команда), Сотрудник (только своя).

## 4.3 Внутренние страницы

### 4.3.1 /decomposition — Список

**Фильтры:**
- Период (текущий месяц / предыдущий)
- Сотрудник (Select или All)
- Роль

**Таблица-обзор:**
- Имя · Роль · KPI% · ФВ Ф/П · Продажи Ф/П · Выручка Ф/П · Рабочих дней Ф/П
- Клик на строку → детальная декомпозиция

### 4.3.2 /decomposition/:employee_id — Декомпозиция сотрудника

**Заголовок:**
- Аватар + Имя + Роль + Месяц
- Кнопки: Редактировать план / Экспорт

**Блок 1: Сводка месяца**
4 карточки:
- KPI % выполнения
- ФВ: факт / план
- Продажи: факт / план
- Выручка: факт / план

**Блок 2: Ежедневная таблица**

Колонки для МП:
- № · Дата · День · Статус явки · ФВ (П/Ф) · Продажи (П/Ф) · Выручка (П/Ф) · БЕЗ НВ (П/Ф) · Прод.НВ · Выр.НВ

Колонки для LMAI:
- № · Дата · День · Статус явки · Обращения (П/Ф) · Лиды (П/Ф) · НВ (П/Ф)

Особенности таблицы:
- Выходные дни по расписанию: затенённые строки
- Ячейки Ф — редактируемые для владельца/РОП (inline edit)
- Ячейки П — только для владельца (edit в Plan Modal)
- Строка "Итого" в футере: сумма всех П и Ф + разница
- Цветовая индикация ячеек: факт > план = зелёный, факт < план = красный

**Блок 3: Рабочий календарь (компактный)**
- Мини-календарь текущего месяца
- Цвет ячейки по типу дня: работал / выходной / больничный / отгул / неявка
- Статистика: отработано X дней, больничных Y, отгулов Z

**Блок 4: Редактор плана (Modal)**
Открывается кнопкой "Редактировать план":
- Поля плана: ФВ на месяц / Продажи / Выручка / БЕЗ НВ
- Рабочие дни рассчитываются автоматически из расписания (read-only)
- Подтвердить / Отмена

## 4.4 Действия пользователя

- Просмотреть план и факт за месяц
- Редактировать план (Владелец, РОП)
- Редактировать ячейки факта вручную (override данных из консультаций)
- Просмотреть детали конкретного дня (→ Записи отфильтрованные по дате)
- Переключиться между сотрудниками
- Переключиться между месяцами

## 4.5 Данные

| Данные | Источник |
|--------|----------|
| Планы | `kpi_plans` |
| Факт ФВ/Продажи/Выручка | авто из `consultations` |
| Ручной override факта | `daily_facts` |
| Статус явки | `attendance` |
| Расписание (выходные) | `schedules` |

## 4.6 Приоритет данных факта

```
Если в daily_facts есть ручной override для дня → использовать его.
Иначе → считать из consultations за этот день.
```

## 4.7 Автоматизация

- Каждую ночь пересчитывается агрегат факта по всем сотрудникам
- При достижении 100% плана → уведомление владельцу
- При отставании >20% к середине месяца → уведомление владельцу и РОП

## 4.8 Связи

- Записи → источник данных ФВ/Продажи
- Рабочий календарь → явки и рабочие дни
- Зарплаты → KPI факт
- Dashboard ← итоговые метрики

# 5. Записи на консультацию

## 5.1 Цель страницы

Полный журнал всех первичных встреч (ФВ) и записей клиентов.
Основная операционная таблица менеджеров.

## 5.2 Доступ

Владелец (все), РОП (все), МП (только свои записи), LMAI (только свои).

## 5.3 Внутренние страницы

### 5.3.1 /consultations — Журнал записей

**Топ-панель:**
- Поиск по имени клиента / телефону / номеру сделки
- Фильтры: Дата · Менеджер · Формат · Статус · После ФВ · Записала · Период
- Кнопка "Добавить запись" (Primary, top-right)

**Таблица записей:**
- Дата · Время · Клиент · Телефон · Формат · Записала · Менеджер · Статус · После ФВ · Сумма · Действия
- Сортировка по дате (новые сверху)
- Пагинация: 25 записей на странице или бесконечная прокрутка
- Клик на строку → Modal редактирования
- Hover → кнопки действий (Редактировать, Удалить)

**Статусы (Status Pipeline визуально):**
```
Придёт → Пришла → Купила/Не купила
     ↘ Не придёт / Отменил / Не отвечает / Перезапись
```

**Строка итогов под таблицей:**
- Всего за период: X записей · Y ФВ · Z продаж · Σ KGS

### 5.3.2 Modal: Добавить / Редактировать запись

**Секция "Клиент":**
- Имя клиента (text)
- Телефон (+996 · автоформат)
- Дата (date picker)
- Время (time picker)
- Формат (Онлайн / Офлайн)
- Номер сделки (digits only, max 7, prefix #)

**Секция "Запись":**
- Записала (буквы только)
- Менеджер (Select из сотрудников)

**Секция "Статусы":**
- Статус планируемый: Придёт / Не придёт / Перезапись / Отменил / Не отвечает
- Фактический: Пришла / Не пришла
- После ФВ: Купила / Не купила / Предоплата / Дожать / Отказ

**Условная логика:**
- "Отказ" → обязательное поле "Причина отказа" (textarea, красный фон)
- "Купила" или "Предоплата" → обязательное поле "Сумма (KGS)"

**Секция "Финансы":**
- Сумма продажи (KGS)
- Расходы доставки (KGS)
- Чекбокс: БЕЗ НВ (клиент пришёл без предварительной записи)

**Секция "Комментарий":**
- Textarea (свободный текст)

## 5.4 Действия пользователя

- Создать запись
- Редактировать любое поле записи
- Удалить запись (с подтверждением)
- Фильтровать журнал
- Искать по клиенту / телефону
- Экспортировать в Excel

## 5.5 Данные

| Данные | Источник |
|--------|----------|
| Записи | таблица `consultations` |
| Менеджеры | `employees` WHERE role IN ('МП', 'LMAI') |

## 5.6 Автоматизация

- При статусе "Купила" или "Предоплата":
  - Автоматически создаётся транзакция дохода в `finances`
  - Факт продаж в `daily_facts` обновляется для менеджера
- При создании записи с датой "сегодня" → появляется в расписании на Dashboard
- Ежедневный агрегат: пересчёт ФВ/Продаж/Выручки для декомпозиции

## 5.7 Связи

- Dashboard ← лента событий из consultations

# 7. Рабочий календарь

## 7.1 Цель страницы

Учёт явок, больничных, отгулов и отпусков каждого сотрудника.
Источник данных для расчёта рабочих дней в Декомпозиции и Зарплатах.

## 7.2 Доступ

Владелец (все), РОП (своя команда), Сотрудник (только свой).

## 7.3 Внутренние страницы

### 7.3.1 /calendar — Обзор команды

**Период:** текущий месяц (переключение по стрелкам)

**Фильтр:** сотрудник (All / конкретный)

**Таблица: Сотрудник × Дни месяца**
- Строки: каждый сотрудник
- Колонки: дни 1–31
- Ячейки: цветной прямоугольник по типу дня

**Типы дней и цвета:**
- Выходной (по расписанию): `#f3f4f6` без контента
- Работал: `#0c4d6c` (синий)
- Больничный: `#fef3c7` (жёлтый)
- Отгул: `#dbeafe` (голубой)
- Неявка: `#fee2e2` (красный)
- Отпуск: `#f3e8ff` (фиолетовый)
- Нет данных: пусто

**Статистика в заголовке строки:**
- Отработано: X дней
- Больничных: Y
- Неявок: Z

**Клик на ячейку:** Popover с выбором статуса дня (только Владелец/РОП)

### 7.3.2 /calendar/:employee_id — Личный календарь

**Полноразмерный месячный вид:**
- Традиционная сетка: 7 колонок (дни недели)
- Каждая ячейка: дата + статус + иконка

**Легенда внизу**

**Кнопка "Отметить день":** Modal с выбором даты + типа + комментария

**Статистика сбоку:**
- Отработано / план
- Больничных / Отгулов / Неявок
- Дней до конца месяца

## 7.4 Действия пользователя

- Отметить тип дня для сотрудника (Владелец, РОП)
- Bulk-редактирование: отметить несколько дней сразу
- Переключение месяца
- Просмотр статистики

## 7.5 Данные

| Данные | Источник |
|--------|----------|
| Явки | таблица `attendance` |
| Расписание (выходные) | `schedules` |
| Список сотрудников | `employees` |

## 7.6 Автоматизация

- Выходные дни по расписанию проставляются автоматически в начале месяца
- При неявке без статуса к 12:00 → уведомление владельцу
- Данные явок → Декомпозиция (рабочие дни) → Зарплаты (расчёт)

## 7.7 Связи

- Декомпозиция ← рабочие дни из attendance
- Зарплаты ← рабочие дни
- Сотрудники → ссылка на личный календарь
